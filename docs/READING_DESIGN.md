# リーディングパート設計 - AiKen (eiken-app)

英検1級〜5級のリーディング問題の傾向・特徴を整理し、アプリのリーディングモジュール構成の検討に使うドキュメントです。

**参照**: [英検 試験内容](https://www.eiken.or.jp/eiken/exam/)、[2024年度 問題形式リニューアル](https://www.eiken.or.jp/eiken/2024renewal/)

---

## 1. リーディングの一般的な3区分

本番の出題は、多くの級で次の3タイプに整理できる。

| 区分 | 内容 |
|------|------|
| **大問1: 短文の語句・空所補充** | 単語・熟語・文法・口語表現の知識。短文または会話の空所に適切な語句を選ぶ。 |
| **大問2: 会話文の空所補充** | 日常会話や文脈に合う「文・語句」を選ぶ。級によっては大問1に含まれる。 |
| **大問3・4: 長文の内容一致・空所補充** | メール・掲示・エッセイ・説明文などを読み、内容理解や空所補充を問う。 |

級が上がるほど「長文の語句空所補充」が独立した大問になり、文量・語彙レベル・論理構造の難度が上がる。

## 2. 級別リーディング構成（2024年度以降）

### 2.1 英検1級（大学上級程度）

- **リーディング時間**: 筆記試験内（リーディング・ライティング合計100分）
- **総問題数**: **35問**（2024年度より41問→35問）

| 大問 | 形式 | 問題数 | 詳細 |
|------|------|--------|------|
| 1 | 短文の語句空所補充 | **22問** | 短文・会話文。高度な語彙・句動詞・イディオム・文法。4肢選択。 |
| 2 | 長文の語句空所補充 | **6問** | パッセージ2本（各3問）。300〜400語程度。接続詞・文の一部など論理的な補充。説明文・評論文。 |
| 3 | 長文の内容一致選択 | **7問** | 長文2題。500〜800語程度。内容に関する質問に4肢から選択。 |

**傾向メモ**  
- 大問1は語彙負荷が最も高く、単語・熟語の知識が直結する。  
- 大問2はディスコースの流れ（因果・対比・言い換え）を理解して空所を埋める形式。  
- 大問3は長文全体の要旨・段落ごとの役割・具体例の把握が求められる。

---

### 2.2 英検準1級（大学中級程度）

- **リーディング時間**: 筆記試験内（リーディング・ライティング合計90分）
- **総問題数**: **31問**（2024年度より41問→31問）

| 大問 | 形式 | 問題数 | 詳細 |
|------|------|--------|------|
| 1 | 短文の語句空所補充 | **18問** | 短文・会話文。準1級レベルの語彙・文法。4肢選択。 |
| 2 | 長文の語句空所補充 | **6問** | パッセージ2本（各3問）。250語程度。述部・名詞句・修飾節・文頭の接続表現など。説明文・評論文。 |
| 3 | 長文の内容一致選択 | **7問** | 長文2題（1本目400語程度・3問、2本目500語程度・4問）。内容質問＋「英文の続き」を選ぶ形式も。 |

**傾向メモ**  
- 大問2では「つなぎ言葉」（ディスコースマーカー）がほぼ毎回出る。  
- テーマは社会・環境・自然・医療・健康など。地域も多様。  
- 内容一致では本文の言い換え選択肢が多く、精読と要約力が重要。

---

### 2.3 英検2級（高校卒業程度）

- **リーディング時間**: 筆記試験内（リーディング・ライティング合計85分）
- **総問題数**: **31問**（2024年度より38問→31問）

| 大問 | 形式 | 問題数 | 詳細 |
|------|------|--------|------|
| 1 | 短文の語句空所補充 | **17問** | 短文・会話文。単語約10問・熟語約7問・文法約3問の割合が多い。4肢選択。 |
| 2 | 長文の語句空所補充 | **6問** | パッセージA・B各3問。説明文で270語程度。接続詞や述部の補充が頻出。 |
| 3 | 長文の内容一致選択 | **8問** | 3A: Eメール約240語・3問。3B: 説明文約350語・5問。ビジネス・環境・医療・歴史など。 |

**傾向メモ**  
- 大問1は比較的得点しやすく、語彙力がものを言う。  
- 大問2は「文脈に合う語句」より、論理マーカー（However, Therefore 等）や文の完結性を問う問題が多い。  
- 大問3はリーディング最難関。選択肢は本文の言い換えなので、パラフレーズ力が鍵。

※2級は公式「試験内容」ページの一次試験表の詳細未取得のため、リニューアル表と一般的な対策情報から整理。

### 2.4 英検準2級（高校中級程度）

- **リーディング時間**: 筆記試験内（リーディング・ライティング80分）
- **総問題数**: **29問**（2024年度より37問→29問）

| 大問 | 形式 | 問題数 | 詳細 |
|------|------|--------|------|
| 1 | 短文の語句空所補充 | **15問** | 短文・会話文。文脈に合う語句を補う。4肢選択。 |
| 2 | 会話文の空所補充 | **5問** | 会話の空所に適切な文や語句を補う。 |
| 3 | 長文の語句空所補充 | **2問** | パッセージ2本（物語文・説明文）。空所に文脈に合う語句を補う。（2024で3問削減） |
| 4 | 長文の内容一致選択 | **7問** | Eメール・説明文。パッセージの内容に関する質問に答える。 |

**傾向メモ**  
- 5級〜3級と同様、短文・会話・長文の流れ。長文語句は「2パッセージで計2問」と少なめ。  
- 話題は学校・趣味・旅行・文化・人物紹介・歴史・教育・科学・自然など。

### 2.5 英検3級（中学卒業程度）

- **リーディング・ライティング時間**: 65分（2024年度より50分から延長）
- **総問題数**: **30問**

| 大問 | 形式 | 問題数 | 詳細 |
|------|------|--------|------|
| 1 | 短文の語句空所補充 | **15問** | 短文・会話文。文脈に合う語句を補う。4肢選択。 |
| 2 | 会話文の空所補充 | **5問** | 会話文の空所に適切な文や語句を補う。 |
| 3 | 長文の内容一致選択 | **10問** | 掲示・案内、Eメール（手紙文）、説明文。パッセージの内容に関する質問。 |

**傾向メモ**  
- 長文は「語句空所」がなく、いきなり「内容一致」のみ。  
- 掲示・Eメール・説明文の3種類で、実用的な読み取りが中心。

### 2.6 英検4級（中学中級程度）

- **リーディング時間**: 35分
- **総問題数**: **35問**

| 大問 | 形式 | 問題数 | 詳細 |
|------|------|--------|------|
| 1 | 短文の語句空所補充 | **15問** | 短文・会話文。文脈に合う語句を補う。4肢選択。 |
| 2 | 会話文の空所補充 | **5問** | 会話の空所に適切な文や語句を補う。 |

**傾向メモ**  
- **語句整序**（並べ替え）が独立した大問として存在。5級にはない。  
- 長文は掲示・Eメール・説明文の3タイプ。場面は家庭・学校・地域・電話・アナウンスなど。

### 2.7 英検5級（中学初級程度）

- **リーディング時間**: 25分
- **総問題数**: **25問**

| 大問 | 形式 | 問題数 | 詳細 |
|------|------|--------|------|
| 1 | 短文の語句空所補充 | **15問** | 短文・会話文。文脈に合う語句を補う。4肢選択。 |
| 2 | 会話文の空所補充 | **5問** | 会話の空所に適切な文や語句を補う。 |

**傾向メモ**

- 単語・熟語・会話表現・文法（現在形・命令文・現在進行形・can など中1範囲）が中心。  
- 場面は家庭・学校・地域・電話など身近な話題。

## 4. アプリで扱う「問題タイプ」の整理案

実装時に問題データやUIを分ける場合のタイプ案。

1. **短文の語句空所補充** … 1〜準2級で出題。級ごとに語彙・文法レベルを変える。  
2. **会話文の空所補充** … 5級〜準2級で独立大問。3級以上は短文に統合されることも。  
3. **語句整序** … 5級・4級のみ。日本文＋語句の並べ替え。  
4. **長文の語句空所補充** … 準2級〜1級。パッセージ＋空所数・文量を級で調整。  
5. **長文の内容一致選択** … 3級〜1級。掲示・Eメール・説明文などジャンルと語数を級で調整。

この区分をベースに、DBスキーマ（問題タイプ・級・大問番号）や出題ロジックを設計するとよい。

|----|:--------:|:--------:|:--------:|:------------:|:------------:|
| 1級 | ○ | — | — | ○ | ○ |
| 準1級 | ○ | — | — | ○ | ○ |
| 2級 | ○ | — | — | ○ | ○ |
| 準2級 | ○ | ○ | — | ○ | ○ |
| 3級 | ○ | ○ | — | — | ○ |
| 4級 | ○ | ○ | ○ | — | ○ |

- 「—」はその級では出題されないため、メニューに表示しない。

| コード | 表示名（例） | 説明 |

## 6. DB構成案

既存の `vocabulary` / `writing_prompts` と同様、**問題は level（級）ごとに保持**し、ユーザー回答は `user_profiles.id` に紐づける。

### 6.1 テーブル一覧

| テーブル | 用途 |
|----------|------|
| `reading_short_questions` | 短文の語句空所補充・会話文の空所補充（1問1行） |
| `reading_word_order_questions` | 語句整序（5級・4級） |
| `reading_passages` | 長文パッセージ（長文語句空所・長文内容一致の「本文」） |
| `reading_passage_blanks` | 長文の語句空所補充用の空所・選択肢（1空所1行） |
| `reading_passage_questions` | 長文の内容一致用の設問（1問1行） |
| `reading_results` | ユーザーごとの回答履歴・正誤（任意・後からでも可） |

### 6.2 reading_short_questions（短文・会話）

短文の語句空所補充と会話文の空所補充を同一テーブルで扱う。`question_type` で区別。

```sql
-- 想定カラム
id          bigserial primary key,
level       text not null,              -- 級。'1kyu'/'jun1kyu'/… または vocabulary と揃えて '1級'/'準1級'/… のどちらでも可
question_type text not null,             -- 'short_fill' | 'conversation_fill'
body        text not null,               -- 英文。空所は '__BLANK__' などのプレースホルダーで表現
choices     jsonb not null,              -- ["A", "B", "C", "D"]
correct_index smallint not null,         -- 0-based
created_at  timestamptz default now()
```

- アプリでは語彙モジュールと揃えて `1級`・`準1級`・… で扱うか、DB のみ短いコード（1kyu）にして変換するかは実装時に統一する。

### 6.3 reading_word_order_questions（語句整序）

```sql
id           bigserial primary key,
level        text not null,              -- '4kyu' | '5kyu'
prompt_ja    text not null,              -- 日本文
words        jsonb not null,              -- ["word1", "word2", ...] 並べ替え対象の語句
correct_order jsonb not null,            -- [2,0,1,3,...] 正解の並び（words のインデックス）
created_at   timestamptz default now()
```

### 6.4 reading_passages（長文の親）

長文の語句空所補充と長文の内容一致の**両方**で使う。`passage_type` で種別を管理。

```sql
id            bigserial primary key,
level         text not null,
genre         text,                      -- 'email' | 'notice' | 'explanation' | 'story' | 'essay' など
passage_type  text not null,             -- 'long_fill' | 'long_content'
body          text not null,             -- 本文。long_fill の場合は空所を '__BLANK__' 等で表現可能
created_at    timestamptz default now()
```

- 語句空所補充: 1パッセージに複数空所 → `reading_passage_blanks` で空所ごとに選択肢を管理。
- 内容一致: 1パッセージに複数設問 → `reading_passage_questions` で設問を管理。

### 6.5 reading_passage_blanks（長文の空所）

長文の語句空所補充用。1パッセージあたり複数行（空所の数だけ）。

```sql
id             bigserial primary key,
passage_id     bigint not null references reading_passages(id) on delete cascade,
blank_index    smallint not null,        -- 何番目の空所か（0-based）
choices        jsonb not null,           -- ["A", "B", "C", "D"]
correct_index  smallint not null,
created_at     timestamptz default now()
```

- 本文 `body` のプレースホルダー順と `blank_index` を一致させる（例: 1番目空所 → blank_index=0）。

### 6.6 reading_passage_questions（長文の内容一致設問）

```sql
id            bigserial primary key,
passage_id    bigint not null references reading_passages(id) on delete cascade,
question_text text not null,
choices       jsonb not null,
correct_index smallint not null,
order_num     smallint not null default 0,  -- 表示順
created_at    timestamptz default now()
```

### 6.7 reading_results（回答履歴・任意）

ユーザーがどの問題にどう答えたかを記録する場合。

```sql
id            bigserial primary key,
user_id       uuid not null references user_profiles(id) on delete cascade,
source_type   text not null,             -- 'short' | 'word_order' | 'passage_fill' | 'passage_content'
source_id     bigint not null,           -- 問題ID or 空所ID/設問ID
user_answer   smallint not null,         -- 選んだ選択肢の index
is_correct    boolean not null,
created_at    timestamptz default now()
```

- 正答率表示・苦手分析・復習用に出題する場合はこのテーブルを利用。

### 6.8 RLS・インデックス

- `reading_*` の**問題テーブル**は全ユーザーが参照可能（SELECT のみ、writing_prompts と同様）。
- `reading_results` は自分の行のみ SELECT/INSERT（user_id = 自プロフィール id）。
- 検索用: `reading_short_questions(level, question_type)`, `reading_passages(level, passage_type)` などにインデックスを張るとよい。

## 7. ページ構成案

### 7.1 ルーティング

| パス | 役割 |
|------|------|
| `/reading` | リーディングトップ。**級セレクト**＋その級で有効なメニュー（問題タイプ別リンク）を表示。 |
| `/reading/short` | 短文の語句空所補充。クエリ `?level=2kyu` または state で級を渡す。 |
| `/reading/conversation` | 会話文の空所補充。同上。 |
| `/reading/word-order` | 語句整序。4級・5級のみ。 |
| `/reading/long-fill` | 長文の語句空所補充。準2級〜1級。 |
| `/reading/long-content` | 長文の内容一致選択。3級〜1級。 |
| `/reading/history` | 回答履歴・正答率（任意）。 |

- ライティングと同様、`/reading` で級を選び、選んだ級に応じて「表示するメニュー」を変える。
- メニュー項目は「問題タイプの表示名」＋リンク（`/reading/short?level=2kyu` など）。級はセレクトの値で統一し、遷移先にクエリまたはコンテキストで渡す。

### 7.2 コンポーネント構成イメージ

```
app/(dashboard)/reading/
  page.tsx                 … トップ。級セレクト + メニュー（ReadingMenu 等）
  short/page.tsx           … 短文語句。級でフィルタして出題
  conversation/page.tsx    … 会話空所
  word-order/page.tsx      … 語句整序
  long-fill/page.tsx       … 長文語句空所（パッセージ表示 + 空所選択）
  long-content/page.tsx    … 長文内容一致（パッセージ表示 + 設問リスト）
  history/page.tsx         … 履歴（任意）

lib/data/
  reading-db.ts            … 各テーブルからの取得・reading_results の保存

lib/constants/
  reading.ts               … READING_LEVELS, PROBLEM_TYPES, getMenuItemsForLevel(level)
```

### 7.3 トップページの挙動

1. 初期表示: プロフィールの目標級があればその級をセレクトの初期値に（ライティングと同様）。
2. セレクトで級を変更するたび、`getMenuItemsForLevel(level)` で「その級で有効な問題タイプ」のリストを取得。
3. リストをメニューとして描画（カードまたはリスト）。各項目は `/reading/{short|conversation|word-order|long-fill|long-content}?level=xxx` へリンク。
4. 5級選択時は「長文の内容一致」は出さない。1級選択時は「会話」「語句整序」は出さない、など。

### 7.4 各問題ページの共通イメージ

- URL または state で `level` を受け取る。
- 該当テーブルから `level` でフィルタして問題を取得（ランダムまたは順番）。
- 1問ずつ表示 → 選択肢クリックで回答 → 正誤表示 → 次の問題、または結果サマリー。
- ログイン時は `reading_results` に保存（任意）。
